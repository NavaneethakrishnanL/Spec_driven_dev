## 2ï¸âƒ£ Understand the Spec (`openapi.yaml`)

All functionality comes from this file:

```yaml
openapi: 3.0.0
info:
  title: User Service
  version: "1.0"
paths:
  /users/{user_id}:
    get:
      summary: Get a user by ID
      parameters:
        - in: path
          name: user_id
          required: true
          schema:
            type: integer
      responses:
        "200":
          description: OK
          content:
            application/json:
              schema:
                $ref: "#/components/schemas/User"
        "404":
          description: User not found

components:
  schemas:
    User:
      type: object
      properties:
        id:
          type: integer
        name:
          type: string
      required: [id, name]
```

This file defines:

* routes
* parameters
* responses
* JSON model shapes

# ğŸ“Œ How Spec-Driven Development Works

### 1. Define the API in `openapi.yaml`

This is the **source of truth**.

### 2. Generate the models

Tools ensure 100% consistency with the spec.

### 3. Implement the API

Implementation MUST follow the spec.

### 4. Test using pytest & spec-based tools

You can also use Schemathesis:

```
schemathesis run openapi.yaml --base-url=http://localhost:8000
```

### 5. Change the spec â†’ regenerate models

The implementation will break if it does not match the updated spec.
This ensures strict contract correctness.

* Simplifies onboarding
* Prevents breaking changes
